## 问题诊断

### 当前错误

```
API调用失败: 400 - {"error":{"code":"InvalidParameter","message":"Image 0 failed: Invalid base64 image url"}}
```

### 根本原因

1. **图片URL格式错误**：豆包API不接受 `data:image/jpeg;base64,{base64_string}` 格式
2. **缺少豆包SDK**：`volcenginesdkarkruntime` 未安装，回退到HTTP方式调用
3. **图片传递方式**：应该优先使用COS URL，而非base64编码

### 错误演进历史

1. 早期：使用 `/chat/completions` 端点 → 400错误
2. 中期：使用错误的模型ID → 404 ModelNotOpen
3. 现在：正确的端点和模型，但图片格式错误 → 400 Invalid base64 image url

## 技术方案

### 修复策略

#### 方案一：安装并使用官方SDK（推荐）

- **优点**：官方支持，稳定性高，自动处理图片格式
- **缺点**：需要安装新依赖
- **实施**：

1. 在 `requirements.txt` 添加 `volcengine-python-sdk[ark]`
2. 重新安装依赖：`pip install volcengine-python-sdk[ark]`
3. SDK会自动处理图片格式转换

#### 方案二：修正HTTP调用格式（备选）

- **优点**：无需新依赖
- **缺点**：需要手动处理图片格式转换，维护成本高
- **实施**：

1. 检查豆包API文档，确认正确的图片格式
2. 可能需要将base64转换为URL或使用其他格式
3. 修改 `_call_via_http` 方法的图片传递逻辑

### 推荐方案：方案一（安装官方SDK）

理由：

1. 官方SDK已经安装在代码逻辑中，只是缺失依赖
2. SDK提供了更好的错误处理和类型提示
3. 代码已经实现了SDK回退逻辑，只需安装依赖即可激活
