## 问题分析

日志显示豆包图片生成 API 返回错误：`MissingParameter: The request failed because it is missing 'content' parameter`

## 核心修复

将 `image_generation_service.py` 中的请求体格式从：

```
{"model": "...", "prompt": "提示词", ...}
```

改为：

```
{"model": "...", "content": [{"type": "text", "text": "提示词"}], ...}
```

## 影响范围

- `generate_image()` 方法 - 文生图
- `generate_from_photo()` 方法 - 图生图

## 修复方案

### 问题根因

豆包图片生成 API (`/contents/generations/tasks`) 使用多模态格式，提示词需放在 `content` 数组中，每个元素包含 `type` 和 `text` 字段。

### 代码修改

**文件**: `c:\Users\zhang\SmartAlbum\backend\app\services\image_generation_service.py`

1. **`generate_image()` 方法** (第173-179行)：

- 将 `"prompt": prompt` 改为 `"content": [{"type": "text", "text": prompt}]`

2. **`generate_from_photo()` 方法** (第301-310行)：

- 同样修改 `content` 格式
- 图生图需要额外添加图片内容项

### 尺寸参数

根据用户提供的示例，`size` 支持预设值如 `"2K"`, `"4K"`, `"1:1"` 等，当前代码已支持。
